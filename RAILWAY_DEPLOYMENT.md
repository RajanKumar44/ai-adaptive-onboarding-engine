# Railway Deployment Guide

Railway is the easiest way to deploy this app - no credit card required, auto-deploys from GitHub, and includes free PostgreSQL database.

## Prerequisites

- GitHub account (free)
- Railway account (free: https://railway.app)
- An API key for one of: OpenAI, Google Gemini, or Anthropic

## Step-by-Step Deployment

### Step 1: Initialize Git & Push to GitHub

```bash
# Navigate to project root
cd c:\Users\Rajan\OneDrive\Desktop\ai-adaptive-onboarding-engine-main

# Initialize Git
git init
git add .
git commit -m "Initial commit: AI Adaptive Onboarding Engine"

# Create new repository on GitHub:
# 1. Go to https://github.com/new
# 2. Name it "ai-adaptive-onboarding-engine"
# 3. Click Create Repository
# 4. Follow the "...or push an existing repository from the command line" instructions

# (Paste commands from GitHub, typically:)
git remote add origin https://github.com/YOUR_USERNAME/ai-adaptive-onboarding-engine.git
git branch -M main
git push -u origin main
```

### Step 2: Deploy on Railway

1. **Go to Railway Dashboard**: https://railway.app/dashboard
2. **Click "New Project"**
3. **Select "Deploy from GitHub"**
4. **Authorize GitHub** - allow Railway to access your repos
5. **Select the repository**: `ai-adaptive-onboarding-engine`
6. **Click "Deploy Now"**

Railway will automatically:
- ✅ Detect the Dockerfile
- ✅ Build the application
- ✅ Provision a PostgreSQL database
- ✅ Deploy the backend and frontend

### Step 3: Configure Environment Variables

After deployment starts, you'll see your project dashboard.

1. **Go to the "app" service** in Railway dashboard
2. **Click "Variables"** tab
3. **Add these environment variables**:

```
POSTGRES_USER=postgres
POSTGRES_PASSWORD=<Auto-generated, copy from DB service>
POSTGRES_HOST=<Auto-generated, from DB service>
POSTGRES_PORT=5432
POSTGRES_DB=ai_onboarding
DEBUG=false
APP_ENV=production
JWT_SECRET_KEY=<Generate a random 32+ character string>
```

**Choose ONE LLM provider and add its key:**

**Option A: OpenAI**
```
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4
```

**Option B: Google Gemini**
```
LLM_PROVIDER=gemini
GEMINI_API_KEY=AIzaSy...
GEMINI_MODEL=gemini-1.5-pro
```

**Option C: Anthropic**
```
LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=sk-ant-...
ANTHROPIC_MODEL=claude-3-sonnet-20240229
```

### Step 4: Get Database Credentials

1. **In Railway dashboard**, click on the **"db" (PostgreSQL) service**
2. **Go to "Variables"** tab
3. **Copy these values** to your app service variables:
   - `POSTGRES_USER` → Copy the username
   - `POSTGRES_PASSWORD` → Copy the password
   - `POSTGRES_HOST` → Copy the host (internal Railway domain)
   - `POSTGRES_PORT` → Usually 5432

### Step 5: Access Your Application

Once deployment completes (watch the logs):

1. **Frontend**: Click the "URL" link for the frontend service
2. **API Docs**: `https://your-app-url.railway.app/api/v1/docs`
3. **Swagger UI**: Available at the above URL

### Step 6: Test the Application

1. Open the frontend URL
2. Register a new account
3. Login
4. Test the onboarding pipeline:
   - Upload a resume (PDF)
   - Add a job description
   - See AI-generated learning recommendations

## Troubleshooting

### Build Fails

Check logs in Railway dashboard:
- Click "Logs" tab
- Look for Python dependency issues
- Common fix: Update `requirements.txt`

### Database Connection Error

- Verify `POSTGRES_HOST` has correct Railway internal domain
- Check `POSTGRES_PASSWORD` is correctly copied
- In Railway app settings, ensure "DATABASE_URL" is not set

### LLM API Not Working

- Verify your API key is correct and not expired
- Check that `LLM_PROVIDER` matches your key type
- Test by uploading a resume and checking backend logs

### Port Issues

Railway automatically assigns ports. The `$PORT` environment variable is set automatically.

## Monitoring

1. **View Logs**: Go to service → "Logs" tab
2. **Monitor Metrics**: Go to service → "Monitoring" tab
3. **Check Issues**: Look for errors in real-time logs

## Scaling (If Needed)

- **Database**: Railway can scale PostgreSQL vertically
- **Backend**: Add more instances or upgrade compute tier
- **Both**: Go to service settings → Instance Type/Size

## Cost

- **Free tier**: Sufficient for testing and light production use
- **Paid tier**: $5/month per service if you exceed free limits
- **Usage**: You're charged for database storage, compute, and bandwidth
